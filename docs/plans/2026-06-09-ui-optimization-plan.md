## 1. 现状分析

### 1.1 当前架构

```
单页应用（无路由，纯 state 驱动）
└── App.tsx
    └── 大屏总览（4 区布局）
        ├── 顶栏: Header（统计 + 时钟）
        ├── 左区: KPI 卡片（4 张纵向）
        ├── 中区: 高德地图 + 图传小窗
        ├── 右区: 无人机状态面板
        ├── 底区: 实时事件流（横向滚动）
        ├── AI 助手 Drawer（侧滑）
        └── 历史查询 Drawer（侧滑）
```

### 1.2 组件清单（21 个）

### 1.3 当前 UI 存在的问题

| 问题 | 严重度 | 说明 |
|------|:--:|------|
| 无路由 | 🔴 | 单页无 URL，无法深链/书签/前进后退 |
| 地图无工具栏 | 🟡 | 缩放/测距/卫星图切换只能靠代码 |
| 事件流信息密度低 | 🟡 | 卡片固定 280px 宽，无法调整 |
| 无事件 Toast 提醒 | 🟡 | 新高危事件无弹窗/声音提示 |
| 面板无折叠 | 🟢 | 左右面板占空间，小屏无法隐藏 |
| 地图每次挂载重建 | 🟡 | Drawer 打开时不触发，但路由切换会销毁 |
| 无键盘快捷键 | 🟢 | 高频操作无快捷入口 |
| 无空/加载/错误状态 | 🟡 | 当前 mock 数据掩盖了这些状态 |
| 响应式未适配 | 🟢 | 仅桌面端可用 |

## 2. 新增页面规划

### 2.1 页面路由树

```
/login          登录页（JWT 认证入口）
/               大屏总览（首页，当前页面升级）
/event/:id      事件详情页（从 Modal 升级为独立页）
/drones         无人机管理页（机队 + 日志 + 维护）
/analytics      数据看板（趋势 + 热力 + 准确率）
/settings       系统设置（用户 + 告警 + 模型配置）
```

### 2.2 各页面详细设计

#### 2.2.1 登录页 `/login`

```

│            🛩️ 低空 AI 巡检平台            │
│                                          │
│     ┌────────────────────────┐           │
│     │  用户名                 │           │
│     ├────────────────────────┤           │
│     │  密码                   │           │
│     ├────────────────────────┤           │
│     │  [ 值班员 / 管理员 ]    │           │
```

## 3. 大屏总览 UI 优化

### 3.1 地图工具栏（浮层）

```
┌──────────────────┐
│  🔍+  放大       │
│  🔍-  缩小       │
│  🛰️  卫星图      │
│  📏  测距        │
│  ⛶  全屏        │
│  📍  定位中心    │
└──────────────────┘
```

叠加在地图左上角（MapLegend 上方或替换），使用 Ant Design `FloatButton.Group`。

### 3.2 实时事件 Toast

高危事件发生时，从右上角弹出 Toast 通知：

```
┌─────────────────────────────────┐
│ 🔴 新高危事件                    │
│ G50 K32+500 拥堵异常 87%        │
│ [查看] [调度🚁] [关闭]           │
└─────────────────────────────────┘
```

使用 Ant Design `notification` + `useEventStore` 订阅。

### 3.3 面板折叠

左右面板各加折叠按钮，收起后仅显示窄条（类似 IDE 侧边栏）。

```
展开: [KPI区 170px] [地图 flex] [无人机区 ~220px]
折叠: [  24px  ] [地图 flex] [  24px  ]
```

### 3.4 键盘快捷键

| 快捷键 | 操作 |
|--------|------|
| `D` | 确认当前选中事件 |
| `F` | 地图全屏 |
| `S` | 打开设置 |
| `Esc` | 关闭 Drawer / Modal |
| `1-3` | 事件筛选：全部/高危/中危/低危 |
| `Space` | 调度无人机（选中事件时） |

### 3.5 事件流紧凑模式

底部事件流增加"紧凑/详情"切换按钮：

```
紧凑模式: 一行一条，仅显示 [等级] [类型] [路段] [时间] [操作]
详情模式: 当前 280px 卡片（含置信度/检测源/状态标签）
```

## 4. 技术架构调整

### 4.1 路由方案

```
src/
├── App.tsx              → ConfigProvider + Layout 壳
├── router.tsx           → React Router v7 路由定义
├── pages/
│   ├── LoginPage.tsx    → /login
│   ├── DashboardPage.tsx → / (当前 App 内容迁移至此)
│   ├── EventDetailPage.tsx → /event/:id
│   ├── DronePage.tsx    → /drones
│   ├── AnalyticsPage.tsx → /analytics
│   └── SettingsPage.tsx → /settings
├── components/          → 保持不变，共享组件
├── stores/              → 保持不变
└── hooks/               → 保持不变
```

### 4.2 跨路由状态保持

```
问题: 从 / 跳转到 /event/123 再返回，地图实例被销毁重建
方案:
  1. 地图实例提升到 Layout 层（不随路由卸载）
  2. 或使用 React Router <Outlet> + keepAlive 策略
  3. 推荐: Layout 持有地图实例 + AMapContainer 通过 portal 渲染
```

### 4.3 需要安装的依赖

```bash
npm install react-router-dom          # 路由
npm install @ant-design/charts        # 图表（数据看板）
npm install @ant-design/cssinjs       # Ant Design 6 样式（已有）
```
