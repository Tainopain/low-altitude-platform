> **版本：** MVP v1.0
> **日期：** 2026-06-08（修订）
> **类型：** 产品方案设计 — MVP 验证版
> **目标：** 1 座机巢 + 2 架无人机 + 4 路摄像头，跑通核心链路，验证价值假设

## 1. 产品概述与定位

**产品名称：** 低空 — 无人机高速智能巡检平台

**一句话定位：** 以无人机为空中感知核心、路边摄像头为地面锚点、AI 视觉分析为检测引擎的高速公路实时风险监测与快速确认平台。

### 1.1 要解决的问题

| 传统高速监控痛点 | 低空平台解决方式 |
|-----------------|:------------------|
| 人工盯屏，事后发现 | AI 自动检测 + 实时告警，从被动到主动 |
| 单一摄像头视角，看不清现场 | 无人机快速抵近，4K 画面远程确认 |
| 偏远路段无覆盖 | 无人机机巢部署，15 分钟可达 |
| 事件靠人判断，响应慢 | AI 秒级初判 + 无人机画面确认，分钟级响应 |
| 误报多，值班员麻木 | 无人机抵近二次确认，减少无效出勤 |

### 1.2 MVP 价值假设

> **核心假设：** 无人机抵近确认能将事件误报率降低 50% 以上，并将确认时间从"派人去现场"的小时级缩短到分钟级。

MVP 只验证一件事：**AI 检测到事件 → 值班员一键调度无人机 → 画面回传确认 → 处置或关闭** 这条链路是否跑得通、是否有价值。

### 1.3 MVP 范围边界

| 维度 | MVP 范围 | 明确不做 |
|------|---------|:---------:|
| 硬件 | 1 座机巢 + 2 架无人机 + 4 路摄像头 | 多机巢、毫米波雷达、移动指挥车 |
| 覆盖 | 试点路段 ~60km（G50 一段） | 全线 1,280km |
| 事件类型 | 5 类（事故/拥堵/障碍物/烟雾/火焰） | 14 类细粒度检测 |
| AI 推理 | 云端推理（视频流回传后分析） | 机载端侧 AI |
| 无人机调度 | 值班员手动一键调度 | 自动起飞、机群协同、接力追踪 |
| 用户角色 | 值班员 + 管理员（2 个） | 飞手、调度决策者 |
| 通知 | 钉钉群消息 | 短信、邮件日报 |
| 部署 | Docker Compose 单机 | K8s 集群、灾备 |

### 1.4 目标用户（MVP）

| 角色 | 职责 | MVP 人数 |
|------|------|:------:|
| 值班监控员 | 监控大屏、确认事件、一键调度无人机、关闭归档 | 1-2 人 |
| 系统管理员 | 配置管理、AI 模型参数调整、用户管理 | 1 人 |

## 5. AI 推理层（MVP）

### 5.1 云端视觉检测

**推理管道：**

```
摄像头 RTSP 视频流 / 无人机回传画面
        │
        ▼
YOLOv8 目标检测（云端 GPU 服务器）
        │
        ▼
置信度过滤（> 0.7）
        │
        ▼
事件生成（标注 GPS 桩号 + 时间戳 + 截图）
        │
        ▼
推送到大屏事件列表
```

**MVP 检测的 5 类事件：**

| 类型 | 说明 | 典型场景 |
|------|------|---------|
| `accident` | 交通事故 | 多车追尾、单车碰撞 |
| `congestion` | 拥堵事件 | 车流停滞、排队超 500m |
| `obstacle` | 障碍物 | 路面抛洒物、掉落货物 |
| `smoke` | 烟雾异常 | 边坡起火、车辆冒烟 |
| `fire` | 火焰检测 | 车辆自燃、路边火灾 |

### 5.2 事件分级引擎

```
风险分 = 事件类型权重 × AI 置信度 × 路段属性系数

事件类型权重: fire=10, accident=7, smoke=5, obstacle=3, congestion=3
路段属性系数: 隧道/桥梁=1.5, 弯道/坡道=1.3, 平直路段=1.0

风险分映射:
  高危: ≥60 → 红色 + 大屏弹窗 + 钉钉推送 + 提示调度无人机
  中危: 30-59 → 黄色 + 平台内展示
  低危: <30 → 蓝色 + 仅记录
```

### 5.3 事件状态流转

```
待确认 ──(值班员确认)──→ 已确认 ──(调度无人机)──→ 无人机抵近中
   │                                                    │
   │                                              ┌─────▼─────┐
   │                                              │ 画面确认   │
   │                                              └─────┬─────┘
   │                                         ┌──────────┼──────────┐
   │                                         ▼          ▼          ▼
   │                                     确认属实    误报排除    需要处置
   │                                         │          │          │
   │                                         │          ▼          │
   └──(误报)──→ 已关闭                       │       已关闭        │
                                             │                    │
                                             ▼                    ▼
                                         处理中              处理中
                                             │                    │
                                             ▼                    ▼
                                         已处理              已处理
                                             │                    │
                                             └────────┬───────────┘
                                                      ▼
                                                   已关闭
```

│  🟡 G50 K7+800 障碍物 中危 85% 待确认  09:16   [调度🚁]     │

**核心交互：**

| 操作 | 行为 |
|------|------|

| 点击地图上无人机图标 | 弹出卡片：编号/电量/任务/位置 |
| 点击事件标记点 | 展示事件详情：类型/置信度/截图/路段信息 |
| 事件确认/关闭 | 行内操作按钮 |

### 7.2 无人机在处置中的角色（MVP）

## 10. 产品路线图

### Phase 1 — MVP（当前，1-2 个月）✅

> **目标：** 跑通核心链路，验证无人机抵近确认的价值假设

| 模块 | 交付内容 |
|------|---------|
| 大屏总览 | 地图 + KPI 卡片 + 事件列表 + 无人机位置图层 |
| 事件检测 | YOLOv8 云端推理，5 类事件 |
| 无人机集成 | 1 巢 2 机，一键调度，基本巡逻 |
| 摄像头接入 | 4 路 RTSP 视频流 |
| 事件闭环 | AI 检测 → 分级 → 推送 → 确认/调度 → 关闭 |
| AI 对话 | DeepSeek V3 单模型 |
| 通知 | 钉钉群消息 |
| 用户 | 值班员 + 管理员，两级权限 |
| 部署 | Docker Compose 单机 + 1 座实体机巢 |

**MVP 成功标准：**
- [ ] 5 类事件 AI 检测准确率 ≥ 80%
- [ ] 高危事件从检测到无人机起飞 ≤ 30 秒
- [ ] 无人机抵达任意试点点 ≤ 15 分钟
- [ ] 系统连续运行 7 天无宕机
- [ ] 值班员可在 2 分钟内完成"看到告警 → 调度无人机 → 画面确认"全流程

### Phase 2 — 扩展（3-6 个月）

> MVP 验证通过后扩展

- 机巢扩容至 3-6 座，覆盖更多路段
- 事件类型扩展至 9 类
- 引入事件分级自动联动（高危自动起飞）
- 多模型 AI 对话支持
- 处置工作流完整闭环
- 短信通知 + 邮件日报

### Phase 3 — 智能化（6-12 个月）

- 全场站 12 座机巢覆盖
- 机载端侧 AI（Jetson Orin）实时推理
- 机群协同自动化（自动接力、智能避障）
- 拥堵预测模型
- 移动端适配

### Phase 4 — 平台化（12 个月+）

- 多区域复制
- 开放 API
- 多品牌无人机/传感器接入
- 数字孪生 + 低空交通管理
