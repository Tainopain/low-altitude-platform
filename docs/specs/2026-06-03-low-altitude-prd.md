## 1. 产品定位：AI 是核心引擎，不是附加功能

**产品名称：** 低空 — AI 驱动无人机高速智能巡检平台

### 2.2 AI 模型详情

| 属性 | MVP 选型 | 说明 |
|------|---------|------|
| 检测模型 | YOLOv8-nano (CPU) → YOLOv8x (GPU) | MVP 先用轻量模型验证，后续升级 |
| 训练数据 | 高速公路场景标注数据集 10K+ | 含白天/夜间/雨天/雾天 |
| 检测类别 | 5 类 (accident / congestion / obstacle / smoke / fire) | 每类 ≥2K 标注样本 |
| 输入分辨率 | 640×640 | 平衡速度与精度 |
| 推理延迟 | < 500ms (GPU) / < 2s (CPU) | 4 路并行 |
| 模型更新 | 月度增量微调 | 基于人工反馈新标注数据 |
| 评估指标 | mAP@0.5 ≥ 0.85, Recall ≥ 0.90 | 高速公路场景 |

### 2.3 AI 置信度校准

AI 输出不仅是检测结果，还有**校准后的置信度**，用于后续分级决策：

| 阶段 | 方法 | 作用 |
|------|------|------|
| 原始输出 | YOLOv8 softmax 概率 | 模型原始置信度 |
| 温度缩放 | Platt Scaling | 校准概率分布，消除过自信 |
| 上下文修正 | 时段×天气×路段属性×连续帧一致性 | 降低夜间误报、提高拥堵置信度 |
| 历史对比 | 与同路段同时段基线对比 | 异常检测增强 |

## 3. AI 决策层 — 智能分级、调度与辅助

### 3.1 AI 事件分级引擎（ML 模型，非规则引擎）

```
输入特征向量 X:
├─ AI 检测置信度 (校准后)
├─ 事件类型 (one-hot 5 维)
├─ 路段属性 (隧道/桥梁/弯道/直道)
├─ 时段 (高峰/平峰/夜间)
├─ 天气 (晴/雨/雾/风)
├─ 历史事件密度 (该路段 7 天内事件数)
└─ 连续帧一致性得分

           ▼
    ┌──────────────┐
    │ XGBoost 分级器 │  ← 轻量级 ML 模型，可解释
    │ 输出: 高危/中危/低危 + 风险分(0-100)
    └──────┬───────┘
           ▼
    分级结果 + 处置建议
```

**为什么用 ML 而不是规则？**
- 规则引擎难以处理特征交互（如：夜间+弯道+烟雾 的组合权重）
- ML 模型可基于历史处置数据持续优化权重
- XGBoost 可解释性强，可输出特征重要性供人工审查

### 3.2 AI 调度决策引擎（Phase 2）

```
调度决策 AI 模型输入:
├─ 事件位置 + 风险等级
├─ 各无人机状态 (位置/电量/续航/任务)
├─ 气象条件 (风速/能见度/降雨)
├─ 空域限制 (禁飞区/限飞区)
└─ 历史飞行数据 (该点飞行次数/电池消耗)

           ▼
    ┌──────────────────┐
    │ 多目标优化         │
    │ · 最短到达时间     │
    │ · 最低飞行风险     │
    │ · 最少电池消耗     │
    │ · 最优航线平滑度   │
    └────────┬─────────┘
             ▼
    推荐无人机 + 最优航线 + 预计到达时间
```

## 8. AI 相关的非功能性需求

### 8.1 AI 性能指标

| 指标 | MVP 目标 | 测量方法 |
|------|:------:|---------|
| 单帧推理延迟 | < 500ms (GPU) | 99th percentile |
| 4 路并行吞吐 | ≥ 8 fps | 持续 1 小时压测 |
| 模型加载时间 | < 10s | 冷启动到首帧输出 |
| GPU 显存占用 | < 8GB | nvidia-smi 监控 |
| 误报率 | ≤ 25% | 每周人工抽检 100 条 |
| 漏报率 | ≤ 15% | 回溯历史录像验证 |
| 对话响应时间 | < 3s | 端到端延迟 |

### 8.2 AI 可用性

| 指标 | 目标 |
|------|:----:|
| 视觉推理服务可用性 | ≥ 99.5% |
| 对话 AI 服务可用性 | ≥ 99% |
| 模型降级策略 | 主模型故障时自动切换备用轻量模型 |
| AI 服务健康检查 | 每 30s 心跳 + 推理质量采样 |

### 8.3 AI 数据安全

| 领域 | 要求 |
|------|------|
| 训练数据存储 | 脱敏后存储，禁止包含车牌/人脸 |
| 模型访问控制 | API Key + IP 白名单 |
| 推理日志 | 保留 30 天，不含原始图像 |
| 模型文件 | 加密存储，签名校验 |

## 9. AI 投资回报（ROI）估算

### 9.1 AI 替代人力估算（试点路段 60km）

| 传统方式 | 人力需求 | AI 替代后 | 节省 |
|---------|:------:|:------:|:---:|
| 人工盯屏 (4 路) | 2 人 × 3 班 = 6 人 | 1 人值班 | **-5 人** |
| 人工巡检 (每日 2 次) | 2 车 × 2 人 = 4 人 | 无人机自动 | **-4 人** |
| 事件确认 (去现场) | 平均 30min/次 | AI+无人机 3min | **10x 加速** |
| 事件分级 | 靠经验，不一致 | AI 一致分级 | **质量提升** |

### 9.2 AI 核心价值指标

| 价值维度 | MVP 预期 | 量化方式 |
|---------|:------:|---------|
| 检测时效 | 事后→实时 | 延迟从小时级降到秒级 |
| 误报降低 | 基线-30% | 人工标注 vs AI 输出的误报率对比 |
| 确认加速 | 30min→3min | 从发警报到画面确认的时间 |
| 知识沉淀 | 0→案例库 | 历史事件处置方案积累量 |

## 10. 产品路线图（AI 视角）

### Phase 1 — MVP（当前，1-2 个月）✅

**AI 交付：**
- [ ] YOLOv8 5 类事件检测 (mAP ≥ 0.80)

- [ ] AI 推理性能监控 Dashboard

- [ ] RAG 知识库上线 (历史案例检索)
- [ ] Text-to-SQL 自然语言查询
- [ ] AI 调度推荐模型 (多目标优化)
- [ ] 主动学习采样策略 (难例优先标注)
- [ ] 月度增量 fine-tune 流水线
- [ ] 多模态感知 (可见光+红外融合)
- [ ] A/B 实验框架 (模型版本对比)

### Phase 3 — AI 自主（6-12 个月）

- [ ] 机载端侧 AI (Jetson Orin + TensorRT)
- [ ] 边缘-云端协同推理
- [ ] 联邦学习 (多路段数据联合训练，隐私保护)
- [ ] 端到端自主巡检 (AI 决策起飞/航线/返航)
- [ ] 预测性维护 (无人机电池/电机故障预测)
- [ ] 拥堵预测模型 (时序 Transformer)

- [ ] 跨场景迁移学习 (高速→城市道路→园区)
